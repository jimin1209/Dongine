> **목적**: 데모 시작 **직전** 1~2분 안에 푸시 알림·위치 공유 지도·Google Calendar 연동이 살아 있는지 빠르게 확인한다.

## 1. 푸시 알림 (FCM)

### 사전 조건

| # | 항목 |
|---|------|
| 1 | Cloud Functions 배포 완료 (`firebase deploy --only functions --project=dongine-13214`) |
| 2 | 기기에서 알림 권한 **허용** 상태 (Android: 설정 → 앱 → 동이네 → 알림, iOS: 설정 → 동이네 → 알림) |
| 3 | 테스트 계정 2개가 **같은 가족**에 소속 |

### 점검 절차 (~1분)

1. **기기 A**에서 계정 1로 로그인, **기기 B**에서 계정 2로 로그인 — 둘 다 같은 가족 확인.
2. 기기 B를 **홈 화면으로 나가거나 다른 탭**으로 이동(백그라운드/포그라운드 모두 가능).
3. 기기 A에서 **채팅 메시지 전송**.
4. 기기 B에 **푸시 알림 또는 포그라운드 스낵바**가 표시되는지 확인.
5. 알림 탭 → 채팅 화면으로 이동하는지 확인.

### 실패 시 먼저 볼 곳

| 증상 | 확인 지점 |
|------|----------|
| 알림이 아예 안 옴 | Firebase Console → Functions 로그에서 트리거 실행 여부 확인 |
| Functions 로그에 에러 없음 | 기기의 FCM 토큰이 Firestore `users/{uid}.fcmTokens`에 저장되어 있는지 확인 |
| 토큰은 있는데 안 옴 | 기기 알림 권한 재확인, iOS라면 APNs 키가 Firebase Console에 등록되어 있는지 확인 |
| "invalid-registration-token" | 앱을 재시작하여 토큰 재등록 후 재시도 |

## 2. 위치 공유 지도 (네이버맵)

### 사전 조건

| # | 항목 |
|---|------|
| 1 | 네이버맵 Client ID가 실제 값으로 입력됨 (`YOUR_NAVER_MAP_CLIENT_ID`가 아닌 값) |
| 2 | 기기 위치 서비스(GPS) **켜짐** |
| 3 | 앱에 위치 권한 **허용** (Android: "항상 허용" 또는 "앱 사용 중 허용", iOS: "항상" 또는 "앱 사용 중") |

### 점검 절차 (~1분)

1. 하단 탭에서 **위치** 화면 진입.
2. 상단 권한 배너가 **정상**(녹색/경고 없음)인지 확인 — 경고가 뜨면 권한 설정 버튼을 눌러 해결.
3. **위치 공유 토글**을 켠다.
4. 지도에 **내 위치 마커**가 표시되는지 확인.
5. (기기 2대) 다른 기기에서도 토글 ON → 지도에 **상대방 마커**가 나타나는지 확인.

### 실패 시 먼저 볼 곳

| 증상 | 확인 지점 |
|------|----------|
| 지도 자체가 안 뜸 (빈 화면/에러) | 네이버맵 Client ID 확인 — `android/gradle.properties`의 `NAVER_MAP_CLIENT_ID` 값이 플레이스홀더가 아닌지 확인 |
| 지도는 뜨는데 마커 없음 | 위치 권한 상태 확인, 기기 GPS가 켜져 있는지 확인 |
| 토글 켰는데 "위치 불가" 라벨 | 앱 설정에서 위치 권한을 "항상 허용"으로 변경 |
| 상대방 마커가 안 보임 | 상대방 기기에서도 위치 공유 토글이 ON인지, 같은 가족인지 확인 |

## 3. Google Calendar 연동

### 사전 조건

| # | 항목 |
|---|------|
| 1 | Google Cloud Console에서 OAuth 클라이언트 설정 완료 (Android SHA + 패키지명, iOS 번들 ID) |
| 2 | 시연용 Google 계정의 **primary 캘린더**에 테스트 일정 1건 이상 존재 |
| 3 | 기기에 해당 Google 계정이 추가되어 있음 |

### 점검 절차 (~1~2분)

1. 캘린더 탭 진입.
2. **Google Calendar 가져오기** 버튼 탭.
3. Google 로그인 팝업 → 계정 선택 → 캘린더 권한 동의.
4. 가져온 일정이 **월간 뷰에 표시**되는지 확인 (일정 이름·날짜 일치).
5. (선택) 앱에서 일정 하나 생성 → **Google Calendar로 보내기** → Google Calendar 앱/웹에서 반영 확인.

### 실패 시 먼저 볼 곳

| 증상 | 확인 지점 |
|------|----------|
| Google 로그인 팝업이 안 뜸 | OAuth 클라이언트 설정 확인 — Android: `google-services.json`의 SHA 핑거프린트가 현재 빌드 서명과 일치하는지 |
| 로그인 후 권한 동의 화면 없이 에러 | Google Cloud Console에서 Calendar API가 **사용 설정**되어 있는지 확인 |
| 동의 후 일정이 안 가져와짐 | 디버그 콘솔에서 `GoogleCalendarService` 관련 에러 확인, primary 캘린더에 해당 기간 일정이 있는지 확인 |
| "access_denied" 에러 | OAuth 동의 화면이 "테스트" 모드라면 시연 Google 계정이 테스트 사용자 목록에 등록되어 있는지 확인 |

## 빠른 참조 — 한눈에 체크

```
[ ] 푸시: 기기 A → 채팅 전송 → 기기 B 알림 수신
[ ] 지도: 위치 탭 → 토글 ON → 내 마커 표시
[ ] 캘린더: Google 로그인 → 일정 가져오기 → 월간 뷰 표시
```

세 항목 모두 통과하면 데모를 시작한다.
