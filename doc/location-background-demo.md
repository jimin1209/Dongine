## 1. 데모에서 말할 수 있는 것 vs 기술적으로 보장되는 것

### 1-1. 시연용 설명(청중용, 짧게)

- 가족이 **위치 공유를 켜 두면**, 앱이 **백그라운드에 있어도**(홈으로 나간 뒤) 일정 간격으로 좌표를 올려 **지도에 반영**할 수 있다.
- **Android**에서는 공유 중 **알림(전면 위치 / 포그라운드 서비스)**이 뜨는 구성이라, 그 알림이 유지되는 동안 갱신이 이어지는 것을 **체감**하기 쉽다.
- **iOS**에서는 백그라운드에서도 계속 갱신하려면 설정에서 위치를 **「항상」** 허용하는 편이 안정적이다.

위 문장은 **「앱 프로세스가 살아 있고, 로그인·가족·공유 ON·GPS ON 등 조건이 맞을 때」**를 전제로 한다. 이 전제를 말로 덧붙이지 않으면 오해가 생긴다.

### 1-2. 실제 구현 한계(검증·내부 공유용)

| 항목 | 내용 |
|------|------|
| **프로세스 전제** | 추적은 `MainShell`이 마운트된 뒤 `familyLocationTrackingBootstrapProvider`에서만 유지된다. **앱 프로세스가 살아 있을 때만** 스트림·Firestore 업로드가 이어진다. |
| **「백그라운드」의 의미** | 여기서는 **홈으로 보내 앱이 백그라운드에 있는 상태**를 말한다. **스와이프로 완전 종료**, OS의 **메모리 정리로 프로세스 종료** 시에는 **갱신이 멈춘다**. |
| **iOS 백그라운드 갱신** | Geolocator `AppleSettings`에서 **`allowBackgroundLocationUpdates`는 위치 권한이 `always`일 때만** 켜진다. **`whileInUse`(앱 사용 중만)** 이면 백그라운드 스트림 갱신은 **기대하지 않는다**. |
| **Android** | `foregroundNotificationConfig`로 **포그라운드 위치 알림**을 붙인다. 알림이 사라지거나 프로세스가 죽으면 갱신이 끊길 수 있다. **「앱 사용 중만」** 권한에서도 알림 유지 동안 갱신이 이어질 **수는** 있으나 **기기·OS마다 다르다**. |
| **업로드 간격** | `AppConstants.locationUpdateIntervalSeconds`(기본 **30초**)로 Firestore 업로드를 스로틀한다. **실시간 매 초**가 아니다. |

**요약**: 「백그라운드에서도 된다」는 **데모용 요약**이고, 「프로세스 생존 + OS 권한 + iOS는 항상(백그라운드까지 검증할 때)」이 **기술 조건**이다.

## 2. 앱이 살아 있는 동안 vs 앱 종료(완전 종료)

<a id="sec-app-lifecycle"></a>

| 상태 | 위치 갱신·업로드 |
|------|------------------|
| **포그라운드** | 공유 ON·권한·GPS 충족 시 스트림 수신 및(간격 제한 후) 업로드 |
| **백그라운드(홈으로 보냄, 프로세스 유지)** | 위와 동일하게 **시도**하나, **iOS는 `always` 권한 없으면** 백그라운드 갱신이 꺼진다. Android는 **알림이 떠 있는 동안** 유리 |
| **사용자가 앱 완전 종료**(예: 멀티태스킹에서 스와이프) | **기대하지 않음** — 스트림·bootstrap이 끊긴다 |
| **OS가 앱을 종료** | 동일하게 **멈춤** |

데모 대본에서는 **「앱을 켜 둔 상태(백그라운드 포함)」**와 **「앱을 완전히 끈 상태」**를 **반드시 구분**해 말한다.

## 3. Android — 권한·설정·배너·알림 (단계별 수동 점검)

<a id="sec-android-location"></a>

단계는 **위에서 아래로** 진행하면 빠진 항목이 적다.

| 단계 | 확인할 것 | 기대 / 비고 |
|------|-----------|-------------|
| A1 | **기기 위치(GPS)** 켜짐 | 꺼져 있으면 상단 배너 **「위치 서비스 꺼짐」**, CTA **「기기 위치 설정 열기」** |
| A2 | 앱 **위치 권한** | 최초에는 보통 **앱 사용 중**부터; 거부 시 **「위치 권한 없음」** 배너 + **앱 설정** CTA |
| A3 | **「앱 사용 중만」** 인 경우 | 배너: **「일부 기기에서는 … 항상 허용이 좋습니다」** + **「앱 위치 권한 설정」** CTA. (코드상 권장 안내; 실제 갱신은 알림 유지 등에 의존) |
| A4 | **「항상 허용」** 인 경우 | 배너 없음, 부제 **「앱이 백그라운드에 있을 때도 … 가능한 권한」** 류의 정상 설명 |
| A5 | 공유 토글 ON 뒤 **알림** | 제목·본문에 앱 이름·**「가족 위치를 공유하는 중입니다.」** 가 보이는지(포그라운드 위치 채널) |
| A6 | 토글 옆 **라벨** | 정상: **「공유 중」** / 권한 부족: **「켜짐(위치 불가)」** / 백그라운드 미충족(예: iOS만 해당 로직이지만 Android에서도 `isBackgroundSharingFullySupported`가 false인 조합): **「공유 중(백그라운드 제한)」** — Android는 주로 **while in use + 기타** 조합에서 배너와 함께 확인 |
| A7 | **백그라운드 갱신**(깊은 검증) | [매트릭스 8-6](./real-device-validation-matrix.md#location-sharing-matrix): 홈으로 보낸 뒤 **30초 이상** → 타 기기에서 마커·시간 갱신 |

**알림 권한**: Android 13+ 에서 알림이 막히면 사용자 경험이 달라질 수 있으므로, 위치와 별도로 **앱 알림 허용**도 확인한다(푸시 매트릭스 §9와 연계).

## 4. iOS — 권한·설정·배너·표시 (단계별 수동 점검)

<a id="sec-ios-location"></a>

| 단계 | 확인할 것 | 기대 / 비고 |
|------|-----------|-------------|
| I1 | **설정 → 개인정보 보호 및 보안 → 위치 서비스** 켜짐 | 꺼지면 앱 배너 **「위치 서비스 꺼짐」** |
| I2 | **앱 항목 → 위치** | **안 함** / **앱을 사용하는 동안** / **항상** 중 하나 |
| I3 | **「앱을 사용하는 동안만」** | 상단: 제목 **「위치: 앱 사용 중만 허용」**, 배너 **「백그라운드에서도 갱신하려면 … 「항상」으로」** + **「설정에서 「항상 허용」으로 변경」** CTA. 토글 라벨은 **「공유 중(백그라운드 제한)」** 일 수 있음 |
| I4 | **「항상」** | 배너 숨김, **「위치: 항상 허용」**, 토글 **「공유 중」** (다른 제한 없을 때) |
| I5 | 백그라운드 위치 사용 시 | 시스템 **파란 막대/위치 표시** 등은 **Apple/OS 정책** — 앱이 켜는 UI가 아니다 |
| I6 | **백그라운드 갱신 검증** | **항상** 허용 전제로 [매트릭스 8-6](./real-device-validation-matrix.md#location-sharing-matrix) 수행. **앱 사용 중만**이면 8-6은 **N/A** 또는 실패로 기록 |

## 5. 화면 요소 빠른 대조표 (배너·토글·하단 안내)

| 요소 | 용도 |
|------|------|
| **상단 배너** (`statusTitle` / `statusSubtitle` / 선택적 `bannerMessage`) | 서비스 OFF·권한 거부·iOS while-in-use·Android while-in-use 권장 등 |
| **토글 옆 라벨** | 꺼짐 / 켜짐(지도 불가) / 켜짐(위치 불가) / 공유 중(백그라운드 제한) / 공유 중 |
| **지도 하단 반투명 안내** | 프로세스 생존·Android 알림·완전 종료 시 중단·iOS 항상 권장 — **기술 한계를 사용자에게 직접 노출** |

수동 점검 시 **배너 문구·CTA 라벨**이 위 표와 같은지 확인하면, OS 설정 변경 후 **앱 복귀 시** `resumed`에서 스냅샷이 갱신되는지까지 이어서 본다.
