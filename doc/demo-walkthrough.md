# 시제품 데모 워크스루 (3–5분)

> 이 문서는 **동이네(Dongine)** 시제품을 처음 보는 청중 앞에서 3–5분 안에 핵심 기능을 보여주기 위한 시연 시나리오다.
> 시연자는 이 문서를 위에서 아래로 따라가면 된다.

---

## 0. 데모 전 준비 (사람이 직접 해야 하는 항목)

아래 항목이 **모두** 완료되어 있어야 시연이 원활하다. 상세 절차는 [manual-build-inputs.md](./manual-build-inputs.md)와 [release-checklist.md](./release-checklist.md)를 참고한다.

### 외부 서비스 설정

| # | 항목 | 확인 방법 |
|---|------|----------|
| 1 | **Firebase 프로젝트** 생성 및 Blaze 요금제 전환 | Firebase Console에서 `dongine-13214` 프로젝트 확인 |
| 2 | **Firebase Auth**에서 이메일/비밀번호 로그인 활성화 | Firebase Console → Authentication → Sign-in method |
| 3 | **Firebase 설정 파일** 3종 배치 | `android/app/google-services.json`, `ios/Runner/GoogleService-Info.plist`, `lib/firebase_options.dart` |
| 4 | **Firestore 규칙·인덱스** 배포 | `firebase deploy --only firestore:rules,firestore:indexes --project=dongine-13214` |
| 5 | **Cloud Functions** 배포 (푸시 알림용) | `firebase deploy --only functions --project=dongine-13214` |
| 6 | **Storage 규칙** 배포 | `firebase deploy --only storage --project=dongine-13214` |
| 7 | **네이버맵 Client ID** 발급 및 입력 | `android/gradle.properties`, `ios/Flutter/Debug.xcconfig` 에 `NAVER_MAP_CLIENT_ID` 값 |
| 8 | **Android 푸시**: FCM 자동 설정 (google-services.json 포함 시 완료) | — |
| 9 | **iOS 푸시**: APNs 키 발급 → Firebase Console 등록 | Firebase Console → Cloud Messaging → APNs |

### 기기 · 빌드

| # | 항목 | 확인 방법 |
|---|------|----------|
| 1 | 시연용 기기(또는 에뮬레이터) **2대** 준비 — 가족 참가 흐름을 보여주려면 2대가 필요 | — |

### 시연 직전

| # | 항목 | 비고 |

| 4 | 화면 미러링 설정 (청중이 볼 수 있도록) | scrcpy, QuickTime 미러링, 프로젝터 등 |

## 1단계: 첫 실행 — 회원가입 & 로그인 *(~30초)*

| 순서 | 동작 | 보여줄 포인트 |
|------|------|--------------|
| 1-1 | 앱 실행 → 온보딩 화면 노출 | 앱 이름 "동이네"와 간단한 소개 확인 |
| 1-2 | **회원가입** (이메일 + 비밀번호) | Firebase Auth 이메일 인증이 즉시 동작하는 것을 보여줌 |
| 1-3 | 로그인 완료 → 가족 설정 화면으로 이동 | 가족이 없는 상태에서는 가족 생성/참가 화면이 나옴 |

**시연 팁**: 시간 절약을 위해 사전 가입한 계정으로 바로 로그인해도 좋다.

## 2단계: 가족 생성 & 참가 *(~40초)*

| 순서 | 동작 | 보여줄 포인트 |
|------|------|--------------|
| 2-1 | **기기 A**에서 "가족 만들기" → 가족 이름 입력 → 생성 | 가족 생성 즉시 완료, 6자리 초대 코드 자동 발급 |
| 2-2 | 초대 코드를 확인 (가족 설정 화면) | 코드 7일 만료 정책 언급 |
| 2-3 | **기기 B**에서 로그인 → "초대 코드로 참가" → 코드 입력 | 참가 즉시 가족 구성원으로 등록됨 |
| 2-4 | 양쪽 기기에서 홈 대시보드 진입 확인 | 같은 가족 데이터가 실시간 동기화되는 것 확인 |

**시연 팁**: 기기가 1대뿐이면 2-3·2-4는 구두로 설명하고 넘어간다.

## 3단계: 데모 데이터 시드 *(~10초, 선택)*

> Debug 빌드에서만 동작한다. 가족 설정 화면에서 **"데모 데이터 채우기"** 버튼을 누르면 할 일·장보기·가계부·일정 샘플이 한 번에 추가된다.

채워지는 데이터 예시:
- 할 일 4건 (우유 사오기, 거실 청소, 병원 예약 확인, 수학 문제집 풀기)
- 장보기 5건 (바나나, 두부, 삼겹살 600g, 우유 1L, 주방세제)
- 가계부 5건 (마트 45,000원, 교통 20,000원 등)
- 일정 4건 (가족 저녁 식사, 결혼기념일, 소아과 방문, 주말 나들이)

이렇게 하면 빈 화면 없이 바로 각 기능의 UI를 보여줄 수 있다.

## 4단계: 채팅 *(~40초)*

| 순서 | 동작 | 보여줄 포인트 |
|------|------|--------------|
| 4-1 | 하단 탭에서 **채팅** 진입 | 실시간 메시지 목록 |
| 4-2 | 메시지 전송 → 상대 기기에서 수신 확인 | Firestore 실시간 동기화, 읽음 처리 표시 |
| 4-3 | **슬래시 커맨드** 시연: `/todo 빨래 개기` 입력 | 채팅에서 바로 할 일이 생성되는 것을 보여줌 |
| 4-4 | (선택) `/expense 점심 12000` 또는 `/cart 식빵` | 채팅 한 줄로 가계부·장보기 연동 |

**핵심 메시지**: "채팅 하나로 할 일, 가계부, 장보기까지 바로 연결된다."

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| 로그인 실패 | Firebase Auth에 이메일 로그인 비활성화 | Firebase Console → Authentication → Sign-in method에서 활성화 |
| 데이터가 동기화되지 않음 | Firestore 규칙 미배포 또는 인덱스 누락 | `firebase deploy --only firestore` 재실행 |
| 푸시 알림 안 옴 | Cloud Functions 미배포 또는 FCM 토큰 미등록 | `firebase deploy --only functions` 후 앱 재시작 |
| 지도 빈 화면 | 네이버맵 Client ID 미설정 | `gradle.properties`, `xcconfig` 확인 |
| 데모 시드 버튼 안 보임 | Release 빌드 | Debug 빌드(`flutter run`)로 실행 |
| 장보기 쿼리 오류 | Firestore 복합 인덱스 미배포 | `firebase deploy --only firestore:indexes` |
