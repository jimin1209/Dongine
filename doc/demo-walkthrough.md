# 시제품 데모 워크스루 (3–5분)

> 이 문서는 **동이네(Dongine)** 시제품을 처음 보는 청중 앞에서 3–5분 안에 핵심 기능을 보여주기 위한 시연 시나리오다.
> 시연자는 이 문서를 위에서 아래로 따라가면 된다.

---

## 3–5분 본 시연 vs 직전 smoke

| 구분 | **직전 smoke** ([demo-smoke-push-map-calendar.md](./demo-smoke-push-map-calendar.md)) | **본 워크스루**(이 문서) |
|------|--------------------------------------------------------------------------------------|-------------------------|
| **시간** | 약 **1–2분** | 앱 시연 본문만 **약 3–5분**(타임라인 하단 참고) |
| **목적** | 커튼 직전 **푸시·지도·(선택) Google**만 통과 여부 확인 | 청중에게 **데이터 흐름·UI**를 스토리로 보여 줌 |
| **데이터** | 같은 가족·로그인 전제; 필요 시 `[DEMO]`만 **초기화** 후 재시드 | **설정** (`/settings`)에서 **데모 데이터 채우기**로 탭을 채운 뒤 단계 진행 |
| **범위** | 채팅 한 줄·지도 탭·(선택) 캘린더 Google 설정만 | 로그인·가족·채팅·할 일·장보기·가계부·캘린더·지도·푸시까지 **순서 시연** |
| **실패 시** | [우회 표](./demo-smoke-push-map-calendar.md#smoke-fallback)로 대본 축소 | smoke에서 이미 우회 방향을 정해 두고, 본 시연에서도 동일 우회를 재사용 |

**당일 권장 흐름**: 문서 1~7번(빌드·smoke)까지 끝낸 뒤 → **8** Debug에서 시드 → **9** 이 워크스루. 자세한 문서 순서는 [README 표](../README.md#시제품-데모-준비--문서-진입-경로).

## 앱이 실제로 열리는 순서 (코드 기준)

시연을 망치지 않으려면 아래 **라우팅**을 알고 있으면 된다 (`GoRouter`, `initialLocation: /splash`).

### 게이트(로그인·가족)

| 상태 | 다음 화면 |
|------|----------|
| 앱 실행 | **스플래시** (`/splash`) — Auth·가족 정보 로딩 |
| 로그인 안 됨 | **온보딩** (`/onboarding`) → 버튼으로 **로그인** (`/login`) |
| 로그인 됨, 가족 없음 | **가족 그룹 설정** (`/family-setup`) — 가족 만들기·초대 코드 참가 |
| 로그인 됨, 가족 있음 | **홈** (`/home`) — `MainShell` 하단 탭 진입 |

- 온보딩은 **첫 실행·로그아웃 후** 등 계정이 없을 때만 거친다. 사전에 만든 계정으로 로그인하면 스플래시 직후 홈 또는 가족 그룹 설정으로 간다.
- **가족 설정·데모 시드/초기화**: **홈** (`/home`) 앱바 우상단 톱니바퀴 → **설정** (`/settings`, 앱바 제목 「설정」) → 목록 하단 **데모 데이터 채우기** / **데모 데이터 초기화**(Debug 전용).

### 하단 탭(`MainShell` 5개)

| 탭 라벨 | 경로 | 앱바 제목·비고 |
|---------|------|----------------|
| 홈 | `/home` | 「동이네」 |
| 채팅 | `/chat` | **현재 가족 이름**(탭 라벨과 다름) |
| 지도 | `/map` | 「**가족 위치**」(탭은 「지도」) |
| 파일 | `/files` | 「파일」 |
| 캘린더 | `/calendar` | 「캘린더」+ 상단 탭 **캘린더 / TODO / 플래너**; Google 연동은 이 화면 앱바 **톱니바퀴**(툴팁 `Google Calendar 설정`) |

### 홈에서 열리는 전체 화면(스택 `push`, 하단 탭 아님)

| 화면 | 경로 | 진입(예) |
|------|------|----------|
| 할 일 | `/todo` | 홈 「남은 할 일」 카드, 「바로가기 → 할 일」, 「오늘의 할 일 → 전체 보기」 |
| 장보기 | `/cart` | 홈 「장보기 남은 항목」 카드, 바로가기 |
| 가계부 | `/expense` | 홈 「이번 달 지출」 카드, 바로가기 |
| 앨범 | `/album` | 홈 바로가기 |
| IoT | `/iot` | 홈 바로가기 |

장보기 화면 앱바 제목은 **「장보기 목록」**이다.

## 핵심 시연 루트(앱 안 한 줄)

**스플래시** → (필요 시) **온보딩·로그인** → (필요 시) **가족 그룹 설정** → **홈** → **설정**에서 **데모 데이터 채우기** → **채팅** (`/chat`) → **홈으로 돌아와** 할 일·장보기·가계부 카드/바로가기로 각 화면 (`/todo` → `/cart` → `/expense`) → 하단 **캘린더** (`/calendar`, 캘린더·플래너 탭) → 하단 **지도** (`/map`) → (기기 2대) **푸시** 확인 → 남는 시간에 **파일**·**앨범** 등.

## 권장 시연 순서 (한눈에)

문서 **번호 순**은 [README 표](../README.md#시제품-데모-준비--문서-진입-경로)와 같다: 빌드 입력 → `preflight` → (권장) Firebase audit → 체크리스트 → (필요 시) Functions 배포 가이드 → **실기기 매트릭스**(시간 있을 때) → **[데모 직전 smoke](./demo-smoke-push-map-calendar.md)** → **Debug 시드** → 본 워크스루.

앱 쪽만 요약하면:

1. **Debug** `flutter run`(시드 버튼은 Debug 전용).
2. **로그인 → 가족** → **설정** (`/settings`)에서 데모 **채우기** 또는 **초기화 → 채우기**(아래 3단계).
3. **채팅** (`/chat`) → **홈**을 거쳐 **할 일·장보기·가계부** (`/todo`, `/cart`, `/expense`) → 하단 **캘린더** (`/calendar`, 일정·플래너).
4. **지도** (`/map`) · **푸시** — smoke에서 통과했으면 짧게만; 실패 시 [우회 표](./demo-smoke-push-map-calendar.md#smoke-fallback).
5. 남는 시간: **파일** (`/files`)·**앨범** (`/album`)·홈 대시보드 등.

## 0. 데모 전 준비 (문서·환경)

워크스루 **본문**은 앱 화면만 다룬다. **시연 시작 전**에는 아래를 한 번에 맞춘다.

1. **문서 순서**: [README — 시제품 데모 준비](../README.md#시제품-데모-준비--문서-진입-경로) 표 **1~7**(빌드 입력·체크리스트·**직전 smoke**).
2. **앱 실행**: Debug `flutter run` — Release에는 데모 시드 UI가 없다.
3. **계정·가족**: 시연용 계정 2개가 **같은 가족**에 들어가 있는지 확인(푸시·채팅 시연용). 기기 **1대**면 2단계·푸시는 구두 우회.
4. **데모 데이터**: **홈** → **설정** → **데모 데이터 채우기**. 이미 `[DEMO]`가 있으면 대화상자 안내대로 **초기화** 후 다시 채우기.
5. **직전 smoke**(앱 켠 직후 1–2분): [demo-smoke-push-map-calendar.md](./demo-smoke-push-map-calendar.md) — 푸시·지도·(선택) Google; 실패 시 **[우회 표](./demo-smoke-push-map-calendar.md#smoke-fallback)**.

- **가족 참가(2단계)** 를 화면으로 보이려면 기기 **2대**. 1대면 구두로 넘긴다.

---

## 1단계: 첫 실행 — 스플래시·온보딩·로그인 *(~30초)*

| 순서 | 동작 | 보여줄 포인트 |
|------|------|--------------|
| 1-1 | 앱 실행 → **스플래시**에서 로그인·가족 정보 확인 | 짧은 로딩 메시지 후 자동 전환 |
| 1-2 | (미로그인 시) **온보딩** — 기능 소개 후 「새 계정으로 시작하기」 또는 「로그인」 | 앱 이름 **동이네**, 가족 허브 컨셉 |
| 1-3 | **회원가입/로그인** (이메일 + 비밀번호) | Firebase Auth 이메일·비밀번호 |
| 1-4 | 로그인 완료 후 **가족이 없으면** **가족 그룹 설정** 화면으로 이동 | 가족이 이미 있으면 **홈**으로 바로 이동 |

**시연 팁**: 시간 절약을 위해 사전 가입한 계정으로 로그인하면 온보딩을 건너뛴다.

---

## 2단계: 가족 생성 & 참가 *(~40초)*

| 순서 | 동작 | 보여줄 포인트 |
|------|------|--------------|
| 2-1 | **기기 A**에서 **가족 그룹 설정** 화면 → **새 가족 만들기** → 가족 이름 입력 → 생성 | 생성 즉시 완료, **6자리** 초대 코드 자동 발급 |
| 2-2 | 초대 코드 확인 — 이후 **홈 → 설정(톱니바퀴)** 에서 가족·초대 코드 관리 | **7일** 만료 정책 언급 (`AppConstants`) |
| 2-3 | **기기 B**에서 로그인 → 같은 화면에서 **초대 코드로 참가** → 코드 입력 | 참가 즉시 같은 가족 구성원으로 등록 |
| 2-4 | 양쪽 기기에서 **홈** 대시보드 진입 확인 | Firestore 실시간 동기화 |

**시연 팁**: 기기가 1대뿐이면 2-3·2-4는 구두로 설명하고 넘어간다.

---

## 3단계: 데모 데이터 시드·초기화 *(~10–30초)*

> **Debug 빌드에서만** 노출된다 (`kDebugMode`). 화면: **홈 → 우상단 설정 → 스크롤 하단**.

| 버튼 | 동작 |
|------|------|
| **데모 데이터 채우기** | 현재 가족에 할 일·장보기·가계부·캘린더 샘플을 한 번에 추가. 성공 시 **대화상자**로 도메인별 건수 요약이 뜬다. 이미 데모가 있으면 **대화상자**로 중복 방지 이유와 「초기화 → 다시 채우기」 순서를 안내한다. |
| **데모 데이터 초기화** | 제목/이름이 **`[DEMO]`** 로 시작하는 문서만 삭제한다. 사용자가 만든 데이터는 건드리지 않는다. 삭제한 항목은 **대화상자**로 도메인별 건수와, 탭에서 확인·다시 채우기 등 **다음 행동**을 안내한다. 삭제할 것이 없으면 그에 맞는 안내 대화상자가 뜬다. |

채워지는 데이터(모든 항목 이름에 `[DEMO]` 접두사가 붙음):
- 할 일 **4건** (우유 사오기, 거실 청소, 병원 예약 확인, 수학 문제집 풀기)
- 장보기 **5건** (바나나, 두부, 삼겹살 600g, 우유 1L, 주방세제)
- 가계부 **5건** (마트 장보기, 지하철 교통카드 충전, 세탁소, 소아과 진료, 문제집 구매)
- 일정 **4건** (가족 저녁 식사, 결혼기념일, 소아과 방문, 주말 나들이 — 오늘 기준 상대 날짜로 생성)

이렇게 하면 빈 화면 없이 각 탭 UI를 바로 보여줄 수 있다.

## 4단계: 채팅 *(~40초)*

| 순서 | 동작 | 보여줄 포인트 |
|------|------|--------------|
| 4-1 | 하단 탭 **채팅** (`/chat`) 진입 — 앱바는 **가족 이름** | 실시간 메시지 목록 |
| 4-2 | 메시지 전송 → 상대 기기에서 수신 확인 | Firestore 실시간 동기화, 읽음 처리 표시 |
| 4-3 | **슬래시 커맨드** 시연: `/todo 빨래 개기` 입력 | 채팅에서 바로 할 일이 생성되는 것을 보여줌 |
| 4-4 | (선택) `/expense 점심 12000` 또는 `/cart 식빵` | 채팅 한 줄로 가계부·장보기 연동 |

**핵심 메시지**: "채팅 하나로 할 일, 가계부, 장보기까지 바로 연결된다."

## 5단계: 할 일 *(~30초)*

| 순서 | 동작 | 보여줄 포인트 |
|------|------|--------------|
| 5-1 | **홈** (`/home`)으로 돌아온 뒤 「남은 할 일」 카드·**바로가기 → 할 일**·「전체 보기」 등으로 **할 일** (`/todo`, 앱바 「할 일」) 진입 | 목록 UI, 시드 데이터 또는 4-3에서 만든 항목 표시 |
| 5-2 | 항목 하나 탭하여 **완료 토글** | 체크 표시 즉시 반영, 상대 기기에도 동기화 |
| 5-3 | **+ 버튼**으로 새 할 일 추가 (담당자 지정, 마감일 설정) | 가족 구성원에게 직접 할당 가능 |

## 8단계: 캘린더 *(~30초)*

| 순서 | 동작 | 보여줄 포인트 |
|------|------|--------------|
| 8-1 | 하단 **캘린더** 탭 진입 | 월간 뷰(TableCalendar), 일정 있는 날짜 표시 |
| 8-2 | 시드된 `[DEMO]` 일정이 보이는 날짜 탭 → 상세 | 유형 필터(식사·나들이·기념일·병원 등) |
| 8-3 | 새 일정 추가 (시간, 유형 선택) | CRUD, Cloud Functions 일정 생성 알림(조건 충족 시) |
| 8-4 | (선택) **플래너** 탭 전환 | 다가오는 일정 리스트 UI |

**Google Calendar**: 앱 내 진입은 **캘린더** 탭 (`/calendar`) 앱바 **톱니바퀴** → 바텀시트 「Google Calendar 설정」(가족 **설정** `/settings`와 다름). 직전 확인은 [smoke — Google](./demo-smoke-push-map-calendar.md#smoke-google). 안 되면 **앱 일정·시드·플래너**만으로 8단계를 진행한다.

## 9단계: 지도(위치 공유) *(~20초)*

하단 네비 라벨은 **「지도」** (`/map`). 화면 앱바 제목은 **「가족 위치」**다.

| 순서 | 동작 | 보여줄 포인트 |
|------|------|--------------|
| 9-1 | 하단 **지도** 탭 (`/map`) 진입 | 상단 **위치 서비스·권한** 상태 배너 |
| 9-2 | **위치 공유** 토글 ON | 네이버맵에 내 마커, Geolocator 스트림 + Firestore 업로드(설정 충족 시) |
| 9-3 | (기기 2대) 상대도 토글 ON 시 가족 마커 | 같은 가족만 표시 |

**이 단계에서 보여 주는 범위(시간 대비)**  
- 기본 시연은 **앱을 포그라운드에 둔 채** 마커·토글·배너만 보여도 목적에 맞다.  
- **백그라운드에서도 계속 갱신된다**고 말하려면 OS 조건을 같이 말해야 한다. **iOS**는 위치 권한이 **「항상」**일 때만 백그라운드 스트림 갱신이 켜지고, **「앱을 사용하는 동안만」**이면 백그라운드에서는 멈추며 화면에 **「항상」으로 변경** 안내가 뜬다. **Android**는 위치 공유 중 **알림(포그라운드 서비스)**이 뜨는 형태라, 홈으로 나가도 프로세스가 살아 있는 한 갱신이 이어질 수 있다(기기·OS 버전에 따라 「항상 허용」을 권장하는 배너가 보일 수 있음).  
- 앱을 **스와이프로 완전 종료**하면 양쪽 OS 모두 갱신이 멈춘다. 데모에서는 **백그라운드(홈 버튼/제스처)**와 **완전 종료**를 섞어 말하지 않도록 한다.

**사전 리허설과 맞추기**: 포그라운드만 확인하는지, 백그라운드 갱신까지 볼지는 [real-device-validation-matrix.md](./real-device-validation-matrix.md) §8(특히 8-5·8-6)과 같은 기대치로 맞춘다. 8-6은 **iOS 「항상」** 전제가 있다.

점검·지도 실패 시 우회는 [smoke — 지도](./demo-smoke-push-map-calendar.md#smoke-map).

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| 로그인 실패 | Firebase Auth에 이메일 로그인 비활성화 | Firebase Console → Authentication → Sign-in method에서 활성화 |
| 데이터가 동기화되지 않음 | Firestore 규칙 미배포 또는 인덱스 누락 | `firebase deploy --only firestore` 재실행 |
| 푸시 알림 안 옴 | Cloud Functions 미배포 또는 FCM 토큰 미등록 | `firebase deploy --only functions` 후 앱 재시작 |
| 지도 빈 화면 | 네이버맵 Client ID 미설정 | `gradle.properties`, `xcconfig` 확인 |
| iOS에서 백그라운드 갱신이 안 됨 | 위치 권한이 「앱을 사용하는 동안만」 | 설정에서 **항상** 허용으로 변경([실기기 매트릭스](./real-device-validation-matrix.md) §8 참고) |
| 데모 시드 버튼 안 보임 | Release 빌드 | Debug 빌드(`flutter run`)로 실행 |
| 데모 채우기 시 「이미 데모 데이터가 있습니다」 대화상자 | 동일 가족에 이전 시드 잔존(중복 방지) | 대화상자 안내 순서대로 **데모 데이터 초기화** 후 다시 채우기 |
| 장보기 쿼리 오류 | Firestore 복합 인덱스 미배포 | `firebase deploy --only firestore:indexes` |
